# Manual Review Checklist

这份清单给你手动复核每条结果时用。建议每条只花 30 到 90 秒，先快速判断，再决定是否深入。

## 1. 快速判断

先问自己这 5 个问题：

1. 这是不是一个真实的人在做真实的东西？
2. 他在解决的问题具体吗？
3. 有用户反馈、作者回复、或站外证据吗？
4. 这个作者是不是持续在做，而不是发一条就结束？
5. 这条内容值得我进一步看、跟、或约聊吗？

## 2. 30 秒快速筛

如果同时满足下面任意 2 到 3 条，可以先保留：

- 有明确产品
- 有明确问题
- 有内测 / 上线 / demo / MVP
- 有评论反馈
- 有作者回复
- 有 TestFlight / GitHub / 官网 / App Store
- 有客户 / 付费 / 留存 / 营收 / 招聘

如果同时满足下面任意 2 条，通常直接降权：

- 卖课 / 收徒 / 陪跑
- 私信我 / 主页看置顶
- 社群引流
- 泛创业观点
- 明显不是 founder 内容
- 关键词误伤

## 3. 详细复核表

### 3.1 问题

- 问题描述是否具体
- 是否是高频问题
- 是否像真实需求而不是概念包装

记录建议：

- `problem_statement:`
- `problem_strength: high / medium / low`

### 3.2 产品

- 是否真的有产品
- 是 idea、MVP、内测、已上线，还是已商业化
- 是否有明确功能和使用场景

记录建议：

- `product_stage: idea / MVP / beta / launched / monetizing`
- `product_clarity: high / medium / low`

### 3.3 用户与反馈

- 有没有评论区真实反馈
- 有没有作者回复
- 有没有用户想要入口 / 功能 / 修复
- 有没有下载、留存、付费等弱数据

记录建议：

- `user_signal: strong / medium / weak`
- `feedback_notes:`

### 3.4 站外证据

- 是否提到 TestFlight
- 是否提到 GitHub
- 是否提到官网 / App Store / waitlist
- 是否有截图、邀请、社群入口

记录建议：

- `external_proof: none / weak / strong`
- `proof_notes:`

### 3.5 Founder Quality

- 是否持续更新
- 是否会解释为什么做这个
- 是否能吸收反馈
- 是否有产品 sense
- 是否像 builder 而不是内容创作者

记录建议：

- `founder_quality: high / medium / low`
- `founder_notes:`

### 3.6 风险

- 是否像卖课 / 引流 / 个人 IP
- 是否是关键词误伤
- 是否像热闹但不商业
- 是否太早、太散、太不稳定

记录建议：

- `risk_level: high / medium / low`
- `risk_notes:`

## 4. 最终结论

建议用固定结论，方便后续复盘。

- `strong_priority`
  - 明显值得约聊
- `watch_closely`
  - 先跟踪，不急着约
- `needs_manual_review`
  - 有信号但争议大
- `content_noise`
  - 明显噪音
- `wrong_target`
  - 不是你要的对象

## 5. 推荐输出模板

你可以按这个格式手动记一条：

```text
title:
author:
problem_statement:
product_stage:
user_signal:
external_proof:
founder_quality:
risk_level:
best_evidence:
next_action:
final_label:
```

## 6. 最值得保留的三类候选

优先保留这些：

- 问题很具体，而且已经有 MVP / 内测 / 上线
- 评论区有真实需求，作者也在认真回复
- 作者连续围绕一个问题域构建，并且有站外证据

## 7. 最常见的误判类型

这些很容易浪费时间：

- “联创”其实是地名 / 园区 / 租房
- “寻找合伙人”其实是生活搭子 / 学习搭子
- “创业者”其实是课程号 / 个人 IP 号
- “增长 / 变现”其实是卖课 / 流量运营内容
- “AI 产品”其实是行业点评，不是自己在做

## 8. 你人工复核时最该做的事

不是把每条都看很深，而是快速决定：

- 值不值得继续看
- 值不值得记下来
- 值不值得约聊
- 值不值得修改机器规则

如果一条内容让你觉得“机器给高了”，最好顺手记下原因。

比如：

- 被哪个关键词误伤
- 缺了什么证据
- 哪类账号总是高分但不值得看
- 哪类帖子其实值得看，但机器给低了

这些就是下一轮规则优化最值钱的材料。
