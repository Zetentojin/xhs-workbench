# Founder Discovery Framework

这份文档定义了 XHS Workbench 当前用于“小红书找 founder / 找早期项目 / 找可投线索”的判断逻辑。

目标不是找“创业内容”，而是找：

- 正在解决真实问题的人
- 正在做真实产品的人
- 已经出现真实反馈的人
- 值得投资人进一步约聊的人

## 1. 核心判断

在小红书上，最有价值的不是“讲创业的人”，而是“在公开环境里暴露出 founder quality 的人”。

所以系统不以“像不像创业帖子”为主要判断，而是以这 5 个问题为主：

1. 这个人是不是在做真实产品，而不是只在讲观点。
2. 他是不是围绕一个具体问题在做，而不是围绕 buzzword 在做。
3. 有没有真实世界反馈，而不是只有流量反馈。
4. 有没有站外可验证痕迹，说明这不是纯内容表演。
5. 这个作者是不是持续在做，而不是偶尔发一条。

系统会混合三类关键词，不再只搜“创业”“投资”“AI”这类宽泛词。

- 公测
- MVP
- demo
- 第一个用户
- 第一个客户
- 用户访谈
- 复盘

### 2.3 问题词

- 拖延 app
- ADHD 工具
- 健康管理 内测
- 睡眠 工具
- 知识管理 上线
- 播客摘要 工具
- 宠物友好 地图

### 2.4 市场验证词

- 产品上线

- 先找“公开构建”，再找“创业叙事”

## 3. 证据层

系统不会只看搜索结果卡片，而是尽量补 4 类证据。

### 3.1 Note Detail

来自 `xhs read`：

- 正文
- 话题标签
- 发布时间
- IP 地
- 更完整的上下文

### 3.2 Comment Evidence

来自 `xhs comments`：

- 用户提需求
- 用户提 bug
- 用户问入口 / 邀请码 / 上线时间
- 作者回复
- 评论区共创迹象

### 3.3 External Proof

系统会特别提取这些站外信号：

- TestFlight
- App Store
- GitHub
- 官网
- waitlist
- 邀请码
- 飞书群 / 微信群 / Discord / Telegram

### 3.4 Author Consistency

来自 `xhs user` 和 `xhs user-posts`：

- 作者 bio
- 作者粉丝规模
- 最近是否持续发 founder / builder 内容
- 最近标题是否长期围绕同一问题域

## 4. 正向信号

这些信号会显著提高投资人视角下的优先级。

### 4.1 Founder / Builder 信号

- 独立开发
- 联创 / 联合创始人
- build in public
- 从 0 到 1
- 做了个
- 上线了
- 内测 / 公测 / 邀测
- demo / MVP
- 冷启动
- 复盘

### 4.2 产品进展信号

- 版本迭代
- 招募内测
- 用户反馈
- 留存
- DAU / MAU
- 第一个用户
- 第一个客户
- 付费用户
- 签约客户

### 4.3 商业验证信号

- 营收
- MRR / ARR
- PMF
- 天使轮 / 种子轮
- 融资复盘
- 招聘 / 找合伙人

### 4.4 高质量帖子特征

- 问题足够具体
- 不是一条内容，而是连续更新
- 评论区有真实需求
- 作者愿意回应和迭代
- 有可验证的站外痕迹
- 作者长期围绕一个问题域输出

## 5. 负向信号

这些内容默认降权，很多会直接进入 `review` 或 `low_signal`。

### 5.1 引流 / 卖课 / 社群信号

- 收徒
- 陪跑
- 资料领取
- 私信我
- 主页看置顶
- 训练营
- 社群
- 变现课
- 一键复刻
- 代运营

### 5.2 泛内容创业信号

- 趋势判断
- 创业观点
- 投资观点
- 行业盘点
- 干货分享
- 认知提升

### 5.3 关键词误伤信号

- 联创园区 / 地名 / 租房
- 同城搭子 / 生活搭子
- 探店 / 美食 / 健身 / 穿搭
- VC 被误解成维生素 / 美妆 / 其他缩写

### 5.4 风险判读

以下内容不一定全是噪音，但需要人工多看一层：

- “找合伙人”但没有明确产品
- “招募内测”但只有社群动作，没有产品证据
- “创业者”标签很强，但帖子内容像个人 IP 经营
- “有增长数据”但没有具体场景、没有用户反馈

## 6. 分层输出

系统当前把结果大致分成四层：

- `priority`
  - 真实问题 + 真实产品 + 明显反馈 / 站外证据 / 连续构建
- `watchlist`
  - 有 founder 信号，但验证程度还不够
- `review`
  - 有些信号，但噪音风险高，需要人工复核
- `low_signal`
  - 内容农场、卖课、误伤、或与目标偏差太大

## 7. 人工复核方法

机器先筛，人工再做二次判断。推荐按这 4 步走。

### 7.1 先看是不是“真在做”

优先确认：

- 有没有产品
- 有没有用户
- 有没有迭代
- 有没有站外证据

如果这 4 个都没有，通常不值得深看。

### 7.2 再看是不是“真问题”

优先确认：

- 问题是不是足够具体
- 是不是高频 / 高频但痛点明显
- 不是单纯情绪表达或泛内容表达

### 7.3 再看 founder quality

优先确认：

- 作者是否持续更新
- 是否愿意回应反馈
- 是否具备产品 sense
- 是否能把用户问题讲清楚

### 7.4 最后看投资相关度

优先确认：

- 有没有商业化空间
- 有没有早期 traction
- 是否值得约聊
- 是否适合现在就跟进，而不是先观察

## 8. 建议的人工标签

你人工复核时，建议统一用下面这些结论，不要每次发明新说法。

- `strong_priority`
  - 值得优先约聊
- `watch_closely`
  - 值得持续跟踪
- `needs_manual_review`
  - 有信号，但争议大
- `content_noise`
  - 明显是噪音
- `wrong_target`
  - 不是你要找的 founder / 项目

## 9. 人工完善时最该补的字段

如果你后面要人工维护结果，最值得补这几项：

- `problem_statement`
  - 他到底在解决什么问题
- `evidence`
  - 文中或评论里最硬的 1 到 3 条证据
- `product_stage`
  - idea / MVP / 内测 / 已上线 / 有收入
- `user_signal`
  - 评论反馈 / 下载 / 留存 / 付费
- `founder_quality`
  - 持续性、响应速度、产品 sense
- `risk`
  - 噪音、伪创业、个人 IP、引流风险
- `next_action`
  - 继续看 / 约聊 / 加入 watchlist / 丢弃

## 10. 这套方法的边界

它适合做线索发现，不适合直接代替投资判断。

局限包括：

- 小红书天然偏内容展示，不等于真实业务
- 评论热闹不等于有商业价值
- 有些好 founder 很低调，不会写明显创业词
- 有些强信号需要站外验证，单靠站内不够

因此这套系统最适合做：

- 找线索
- 排序
- 缩小人工阅读范围
- 找“值得进一步研究的人”

不适合直接做：

- 投资结论
- 商业尽调
- 团队判断的最终依据
